API Documentation

The HTTP API takes GET requests in the following scheme:

http://geoip.cf/api/{ip}/{type(s)}

You can also those to select all types with the below method:

http://geoip.cf/api/{ip}

IP

The IP address you would like information from. This must be a qualities domain name, or the lookup won't succeed, or will throw an error.

Type

What type of data you would like to return, separated by commas. Leave empty to return all. Types of things that can be returned

The HTTP API will return application/json MIME based responses. If only one type is selected, the API will return text/html basic responses only with the information requested.

Example of fullresponse:

[~]$ curl -sIX HEAD geoip.cf/api/68.32.144.33                                  
HTTP/1.0 200 OK
Content-Type: application/json
Content-Length: 216
Access-Control-Allow-Origin: *
Server: Werkzeug/0.9.6 Python/2.7.3
Date: Sun, 08 Mar 2015 04:51:48 GMT

Example of type-specific response:

[~]$ curl -sIX HEAD geoip.cf/api/68.32.144.33/country                          
HTTP/1.0 200 OK
Content-Type: text/html; charset=utf-8
Content-Length: 13
Access-Control-Allow-Origin: *
Server: Werkzeug/0.9.6 Python/2.7.3
Date: Sun, 08 Mar 2015 04:54:58 GMT

As you can see, the only real difference is the Content-Type. All methods support using Access-Control-Allow-Origin: *, is if in any event you would like to implement the information into a jQuery plugin/website, there should be no Ajax restrictions.

Types

Type Description
country

Country the IP is originating from. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/country
United States
Example
country_abbr

Abbreviated version of the country. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/country_abbr
US
Example
continent

Abbreviated continent. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/continent
NA
Example
subdivision

List of subdivisions (City, State, etc) sorting by accuracy from left to right. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/subdivision
New Jersey
Example
latitude

Latitude. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/latitude
39.8018
Example
longitude

Longitude. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/longitude
-75.008
Example
timezone

Country/State normalized timezone specification. Empty if unknown.

[~]$ curl -s geoip.cf/api/68.32.144.33/timezone
America/New_York
Example
accuracy

The higher the number, the more accurate the results are.

[~]$ curl -s geoip.cf/api/68.32.144.33/accuracy
4
Example
hostname

Hostname of the lookup if available.

[~]$ curl -s geoip.cf/api/68.32.144.33/hostname
c-68-32-144-33.hsd1.nj.comcast.net
Example
ip

IP of the lookup. Redundant if you used an IP for the lookup.

[~]$ curl -s geoip.cf/api/c-68-32-144-33.hsd1.nj.comcast.net/ip
68.32.144.33
Example

Examples

An example full-API response:

[~]$ curl -s geoip.cf/api/68.32.144.33
{
  "accuracy": 4,
  "continent": "NA",
  "country": "United States",
  "country_abbr": "US",
  "hostname": "c-68-32-144-33.hsd1.nj.comcast.net",
  "ip": "68.32.144.33",
  "latitude": 39.8018,
  "longitude": -75.008,
  "subdivision": [
    "New Jersey"
  ],
  "timezone": "America/New_York"
}

Combining multiple types into one query:

[~]$ curl -s geoip.cf/api/68.32.144.33/latitude,longitude                      
39.8018|-75.008

As you can see above, we're separating multiple types with | mainly due to the possibility of one of the types returning , in it's string.
